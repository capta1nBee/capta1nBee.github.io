Platform Roadmap

Platform Yol Haritası

A modular breakdown of the Poyraz-K8s intelligence and security components.

Poyraz-K8s zeka ve güvenlik bileşenlerinin modüler dağılımı.

🚀 AppCreator

🚀 AppCreator

8-step wizard to build K8s apps from scratch. Zero YAML knowledge required. Integrated GitOps flow for PR-based deployments.

Sıfırdan uygulama oluşturmak için 8 adımlı sihirbaz. YAML bilgisi gerektirmez. PR tabanlı dağıtımlar için entegre GitOps akışı.

🛡️ Runtime Security

🛡️ Runtime Güvenliği

eBPF-based real-time syscall monitoring. Detect drift, unauthorized executions, and file access violations instantly.

eBPF tabanlı gerçek zamanlı sistem çağrısı izleme. Kaymaları, yetkisiz çalıştırmaları ve dosya erişim ihlallerini anında tespit edin.

🕸️ Network Monitor

🕸️ Ağ İzleyici

Deep packet analysis at L3/L4/L7. Visual topology mapping of microservice flows with automatic NetworkPolicy generation.

L3/L4/L7 katmanlarında derin paket analizi. Otomatik NetworkPolicy oluşturma ile mikro hizmet akışlarının görsel topoloji haritalaması.

🤖 AI Analyst

🤖 Yapay Zeka Analisti

Integrated LLM (Ollama) that consumes cluster logs and metrics to provide root-cause analysis and health insights.

Kök neden analizi ve sağlık öngörüleri sağlamak için küme günlüklerini ve metriklerini tüketen entegre LLM (Ollama).

Platform Deployment

Platform Kurulumu

Deploy the core Poyraz-K8s control plane using Docker Compose for local or single-instance setups.

Yerel veya tek örnekli kurulumlar için Docker Compose kullanarak çekirdek Poyraz-K8s kontrol düzlemini kurun.

DOCKER COMPOSE
# Clone and build the full stack
git clone https://github.com/poyraz/poyrazk8s.git
cd poyrazk8s
docker-compose up -d --build

# Backend will be available at http://localhost:8686
# Frontend will be available at http://localhost:3000

Hardware Requirements

Donanım Gereksinimleri

Requires Linux Kernel 5.4+ with BTF support enabled for eBPF CO-RE capabilities.

eBPF CO-RE yetenekleri için BTF desteği etkinleştirilmiş Linux Kernel 5.4+ gerektirir.

KUBECTL
# Deploy the security stack
kubectl apply -f runtimesecurity/kubernetes/configmap.yaml
kubectl apply -f runtimesecurity/kubernetes/rules-configmap.yaml
kubectl apply -f runtimesecurity/kubernetes/daemonset.yaml

Flow Architecture & Visualization

Akış Mimarisi ve Görselleştirme

Flows are captured in the kernel using kprobes/uprobes, enriched with K8s metadata (Pod/NS), and aggregated into a dynamic topology map on the UI.

Akışlar, kprobes/uprobes kullanılarak çekirdekte yakalanır, K8s meta verileriyle (Pod/NS) zenginleştirilir ve arayüzde dinamik bir topoloji haritasında birleştirilir.

Role-Based & Cluster-Aware Authorization

Rol-Tabanlı ve Küme-Duyarlı Yetkilendirme

Poyraz-K8s uses a Multi-Tenant Cluster-Aware logic. Permissions are tied to the cluster_uid, ensuring users only see resources in clusters they are authorized for.

Poyraz-K8s, Çok Kiracılı Küme-Duyarlı bir mantık kullanır. Yetkiler cluster_uid'ye bağlıdır, bu da kullanıcıların yalnızca yetkili oldukları kümelerdeki kaynakları görmesini sağlar.

Casbin Integration

Casbin Entegrasyonu

Uses Casbin (e.g. p, admin, cluster-123:namespace-a, view) to manage granular access policies.

Ayrıntılı erişim politikalarını yönetmek için Casbin (örneğin p, admin, cluster-123:namespace-a, view) kullanır.

Exec Security & Terminal Audit

Terminal Güvenliği ve Denetimi

Role-Based Command Permissions

Rol-Tabanlı Komut Yetkileri

Restrict terminal power by assigning RegEx whitelists to roles. For example, Developer roles may run ls or cat, but are blocked from rm -rf or curl.

Rollere RegEx beyaz listeleri (whitelist) atayarak terminal yetkilerini kısıtlayın. Örneğin, Geliştirici rolleri ls veya cat çalıştırabilir, ancak rm -rf veya curl komutları engellenir.

Session Recording (Video Audit)

Oturum Kaydı (Video Denetimi)

Every terminal session is recorded as a series of time-stamped terminal events using rrweb. Admin users can "play back" the session like a video to audit user behavior exactly as it happened.

Her terminal oturumu, rrweb kullanılarak zaman damgalı terminal olayları dizisi olarak kaydedilir. Yönetici kullanıcılar, kullanıcı davranışlarını tam olarak gerçekleştiği gibi denetlemek için oturumu bir video gibi "tekrardan oynatabilir".

Forensic Discovery: Recordings capture shell escapes and hidden inputs, providing 100% visibility into interactive sessions.
Adli Keşif: Kayıtlar, kabuk kaçışlarını ve gizli girişleri yakalayarak etkileşimli oturumlara %100 görünürlük sağlar.
